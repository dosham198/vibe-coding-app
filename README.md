# Vibe Coding Group Challenge App

Cursor를 활용한 Vibe Coding 방식으로 문제 정의, 서비스 기획, 화면 설계, 기능 단위 개발, 외부 서비스 연동 확인, 배포 및 트러블슈팅을 진행한 웹 애플리케이션 프로젝트입니다.
- Google Cloud Console을 활용한 외부 API 연동 설정
- API Key 및 환경 변수 설정 확인
- 배포 환경에서 API 호출이 정상적으로 동작하지 않는 문제를 확인하고 수정
- vercel을 통한 배포

## 1. 프로젝트 진행 과정

먼저 해결하고 싶은 문제를 직접 정의했습니다.
* 은둔·고립 청년 문제
* 일상 활동을 함께할 지지체계 부족
* 작은 성취를 기록하고 공유할 기회의 부족

이를 바탕으로 사용자가 그룹에 참여해 함께 루틴을 수행하고, 인증과 댓글, 보상 기능을 통해 성취를 공유할 수 있는 그룹 루틴 챌린지 앱을 기획했습니다.

## 2. Vibe Coding 활용 방식

Cursor에게 만들고 싶은 서비스의 목적, 타겟 사용자, 핵심 기능을 설명하고 프로젝트 기획과 설계를 요청했습니다.
이 과정에서 다음과 같은 산출물을 생성하고 참고했습니다.

* 프로젝트 기획안
* 화면 설계서 Wireframe
* 디자인 가이드
* 구조 설계 문서 Architecture
* 기능별 개발 To-do


## 3. 개발 과정에서 확인한 내용

프로젝트 진행 과정에서 다음과 같은 웹 애플리케이션 개발 요소를 다뤘습니다.

* Next.js 기반 프로젝트 구조 확인
* 페이지 및 컴포넌트 구조 수정
* 로그인 / 로그아웃 흐름 확인
* Firebase 등 외부 서비스 연동 설정 확인
* API 요청 및 응답 흐름 확인
* 데이터베이스 연결 및 Prisma 관련 설정 확인
* 환경 변수 설정 확인
* Vercel 배포 및 배포 환경 오류 수정

## 4. 주요 기능

프로젝트에서 구현 또는 구성한 주요 기능은 다음과 같습니다.

* 회원가입 및 로그인 / 로그아웃
* 그룹 생성 및 참여
* 루틴 인증
* 댓글 기능
* 신고 기능
* 포인트 및 배지 기반 보상 기능
* 프로필 및 설정
* 취미공유방
* 지나온 길 기능

## 5. 사용된 기술 및 도구

아래 기술들은 프로젝트 코드와 설정 파일에 포함되어 사용된 기술입니다.

* Cursor
* Next.js
* TypeScript
* React
* Tailwind CSS
* Firebase
* Prisma
* PostgreSQL
* Vercel

※ 본 프로젝트는 Cursor를 활용한 Vibe Coding 방식으로 진행되었으며, 일부 코드와 설정은 AI가 생성한 결과물을 바탕으로 실행·수정·검토했습니다.

## 6. 프로젝트 구조

```text
├── app/              # Next.js App Router 페이지 및 API Route
├── components/       # React 컴포넌트
├── lib/              # 인증, DB, 외부 서비스 관련 설정
├── prisma/           # Prisma 스키마 및 DB 관련 파일
├── public/           # 정적 파일
└── docs/             # 기획, 설계, 와이어프레임 문서
```

## 7. 배운 점

Vibe Coding을 통해 AI 코딩 도구를 사용할 때 중요한 것은 단순히 코드를 생성하는 것이 아니라, 사용자가 문제를 명확히 정의하고 AI에게 적절한 단위로 작업을 요청하는 것임을 경험했습니다.

또한 AI가 생성한 코드도 실제 실행, 인증 설정, API 흐름, 데이터베이스 연결, 배포 환경 설정을 거치며 계속 검토하고 수정해야 한다는 점을 확인했습니다.
