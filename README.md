# Vibe Coding Web App

AI 코딩 도구 Cursor를 활용해 기획부터 설계, 구현, 수정, 배포까지 진행한 Vibe Coding 기반 웹 애플리케이션 프로젝트입니다.

본 프로젝트는 단순히 AI가 생성한 코드를 사용하는 것이 아니라, 사용자가 먼저 문제를 정의하고 해결 방향을 설정한 뒤, Cursor를 활용해 기획 문서, 화면 설계, 구조 설계, 기능 구현, 오류 수정 과정을 반복하며 개발한 프로젝트입니다.

## 1. Problem Definition

본 프로젝트는 청년층이 겪을 수 있는 일상 속 고립감과 성취감 부족 문제에서 출발했습니다.

정의한 핵심 문제는 다음과 같습니다.

- 은둔·고립 청년: 사회적 연결이 부족하거나 일상적으로 고립감을 느끼는 청년
- 지지체계 부족: 꾸준한 활동을 함께할 동료나 지지 네트워크의 부재
- 성취감 부족: 작은 성취를 기록하고 인정받을 수 있는 기회의 부족

## 2. Solution

위 문제를 해결하기 위해 그룹 기반 루틴 챌린지 플랫폼을 기획했습니다.

사용자는 그룹을 만들거나 참여하여 함께 루틴을 수행하고, 인증과 댓글을 통해 서로의 활동을 확인하며, 보상 시스템을 통해 작은 성취를 시각적으로 확인할 수 있습니다.

핵심 가치 제안은 다음과 같습니다.

- 함께하는 성취: 혼자가 아닌 그룹과 함께 루틴을 수행하여 지속성 확보
- 시각적 성취 기록: 지나온 길 기능을 통해 자신의 활동 여정 확인
- 자유로운 커뮤니티: 취미공유방을 통한 자연스러운 소통
- 공정한 참여 환경: 신고 시스템을 통한 건전한 챌린지 문화 유지

## 3. Vibe Coding Process

이 프로젝트는 Cursor를 활용한 Vibe Coding 방식으로 진행했습니다.

### 3.1 문제 정의 및 서비스 방향 설정

먼저 해결하고 싶은 문제를 직접 정의하고, 타겟 사용자를 설정했습니다.  
이후 그룹 기반 루틴 챌린지 플랫폼이라는 서비스 방향을 정하고, 필요한 핵심 기능을 구상했습니다.

### 3.2 AI를 활용한 기획 문서 생성

Cursor에게 만들고 싶은 플랫폼의 목적, 타겟 사용자, 핵심 기능을 설명하고 다음 문서 생성을 요청했습니다.

- 프로젝트 기획안
- 개발 계획
- 화면 설계서 Wireframe
- 디자인 가이드 Design Guide
- 코드 구조 설계 Architecture

이를 통해 프로젝트의 전체 구조를 먼저 잡고, 구현해야 할 기능을 섹션별 작업 단위로 나누었습니다.

### 3.3 기능 단위 개발

생성된 기획 문서와 구조 설계를 바탕으로 기능별 To-do를 나누고, Cursor를 활용해 개발을 진행했습니다.

주요 구현 기능은 다음과 같습니다.

- 사용자 인증
- 그룹 생성 및 참여
- 루틴 인증
- 댓글 및 소통 기능
- 신고 기능
- 보상 시스템
- 프로필 및 설정
- 취미공유방
- 지나온 길 기능

### 3.4 코드 검토 및 트러블슈팅

AI가 생성한 코드를 그대로 사용하는 것이 아니라, 실행 과정에서 발생하는 오류를 확인하고 수정하는 방식으로 개발을 진행했습니다.

특히 다음과 같은 부분을 확인하며 수정했습니다.

- 페이지 이동 및 UI 동작 오류
- 인증 관련 오류
- 데이터베이스 연결 및 Prisma 설정 관련 오류
- API 요청 및 응답 처리 오류
- Vercel 배포 환경에서 발생한 환경 변수 설정 문제

### 3.5 배포

프로젝트는 Vercel을 통해 배포했습니다.  
배포 과정에서 로컬 환경과 배포 환경의 차이를 확인하고, 환경 변수와 데이터베이스 연결 설정을 점검했습니다.
